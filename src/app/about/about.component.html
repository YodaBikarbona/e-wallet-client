keyboard_arrow_right News and information
Users can see all news and information about things that changed, added or removed. This section is loaded by default after login. To get back on news and information user needs click on E-Wallet (on menu).
keyboard_arrow_right Profile account_circle
Users can see personal information used for personal reports, and next features.
adjust Change password
Users can change password using combination of upper, lower letters, special symbols, numbers, and min 8 symbols e.g. "iwe56Jot.". To change password through profile user needs know old password. After changing password user will be inactive and needs use an activation code sent on email.
adjust Edit profile
Users can edit their profiles through edit form. Edit form will allow changing any attributes except city and country (User can ask administrator to change current city and country only if user doesn't have any bills). Default currency will help users to adding bills faster (If user has a default currency, then all new bills will select default currency and user can change user currency whenever wants. Categories, subcategories and currencies are information that calculated automatic when user add some category, subcategory or currency inside options.
keyboard_arrow_right Bills list_alt
Users can see all their costs and profits. Using filters users can get more precise results.
adjust Bill Type: Costs
Users can switch bills view using button switch swap_vert . After users picked bills type, all action will be connected with picked type e.g. adding bills, printing reports, seeing table of bills.
adjust Filters
Users can filter data by Category, Subcategory, Currency, Date and Location. After picking Category user can pick only Subcategory that is connected wit picked Category. Also user can pick only Category and Currency that is saved in users categories or user currencies (check options). Filtering by date (from, to) users can see all bills from picked date for filter from date (included), or to date for filter to date (not included). E.g. if user want see all bills for some specific month then user need pick first day in filter from date and first day in filter to date.
adjust Search
Users with search can get more precise than using only filters. Searching is allowed by title.
adjust New bill
To add a new bill user need fill all required fields Category, Currency (can be default), Title, Price and Quantity (1 by default). Also users can set a comment to describe their bills e.g. (I bought a new car for 10000€ in München when I visited my friend..). Manipulation with date is not allowed (when bill is added that will be a bill date). Users also can use "Not My City" choice (turned of by default), this attribute will separate your living city costs/profits of costs/profits that aren't created in your living city or country. Attribute Quantity will be useful if user want se all increasing or decreasing of price for specific category/subcategory. E.g. user added some bill 01.10.2019 and he paid 50.00€ for 26.00l of fuel (diesel), after that user added an another bill 01.01.2020 and he paid 50.00€ for 24.18l of fuel (diesel) and as we can see, the price of fuel (diesel) increase for 0.1447€ per liter. Users can make some research using their own data.
adjust Show info
Using show info open_in_new on results table user can view all details of specific bill, also user can edit and delete bill. Edit bills work same just like adding bills, all required fields need be filled. User can change all attributes except date.
adjust Print report
Using print report picture_as_pdf users can get pdf report on their device. All report information will be filtered by filters.
keyboard_arrow_right Graph equalizer
Users can see graphs (line and pie) to get information in which period user spends most money. This section is useful to get information which period is critical and help user to decrease own costs. In this section user has two filters (currency and date) that work exactly same like bills filters. Line graph will be able to see only if user has at least two bills with different date e.g. user added bill for fuel category on 01.01.2020 and user added another bill for fuel category on 05.01.2020. Line graph will show state from 01-05 (including both if user doesn't use date filter), that means there will be 5 points on graph. Days that user didn't add any bills will be defined by value 0. Pie graph (4 of them) will be able to see if user has at least one bill with some category or subcategory.
keyboard_arrow_right Options settings
In this section users can add categories, subcategories and currencies that want to use. To add some specific subcategory user first needs add category that subcategories belong. User can add max 10 currencies, also can add currency monthly limit (one of next features will send news when user is to close to monthly limit or when user exceeded monthly limit. To delete specific category, subcategory or currency, user needs first delete or change all bills that use that category, subcategory or currency. User can search specific category, subcategory or currency depends on status of attributes. Status could be active or inactive. To delete specific category, subcategory or currency user needs just click on circle within name of specific attribute.
keyboard_arrow_right Application application
In this section users can reports application bugs or suggests some suggestions, rate application and see how application works. How application grows there will be the new sections about application.
